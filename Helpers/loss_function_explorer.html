NLL expects log-probabilities (from log_softmax). CE expects raw logits and applies log_softmax internally. They are mathematically identical when used correctly — CE = NLL(log_softmax(logits)).
CE == NLL(log_softmax(logits)) ✓ Use CE → pass raw logits Use NLL → pass log_softmax output
CE vs NLL — same curve
Both produce the same loss values. NLL exists because sometimes log_softmax is precomputed (e.g. for beam search) and you don't want to recompute it.
Quick reference: Q=0.99 → loss ≈ 0.01 (confident, correct) Q=0.50 → loss ≈ 0.69 (uncertain) Q=0.10 → loss ≈ 2.30 (confident, wrong) Q=0.01 → loss ≈ 4.60 (very wrong)
